# TrainAudit Docs

This directory contains all planning, architecture, and implementation documentation for TrainAudit.

## Structure

```
docs/
├── README.md                    # This file
├── roadmap.md                   # Milestone-by-milestone build plan
├── architecture/
│   ├── overview.md              # System architecture overview
│   └── design-system.md        # UI component and styling approach
├── decisions/
│   ├── 0001-monorepo-adonis-api-tanstack-start.md
│   └── 0002-shadcn-ui-component-foundation.md
└── implementation-steps/
    └── 0000-project-planning.md
```

## How to Read These Docs

**Start with** `roadmap.md` for the big picture.

**Architecture docs** explain the system design and key technical choices.

**Decision records** (`decisions/`) document why important choices were made, what alternatives were considered, and what the consequences are.

**Implementation steps** (`implementation-steps/`) are written after each step is built. They document what changed, why, and how to verify it.
